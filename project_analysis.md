АНАЛИЗ ПРОЕКТА ERP-LITE

ТЕХНОЛОГИЧЕСКИЙ СТЕК

Серверная часть (Backend):
- Язык программирования: Python 3.10+
- Веб-фреймворк: FastAPI
- ORM (уровень данных): SQLAlchemy
- Валидация данных: Pydantic v2
- База данных: SQLite (через aiosqlite для асинхронности)
- Безопасность: JWT (JSON Web Tokens) для авторизации, Bcrypt для хеширования паролей
- Сервер приложений: Uvicorn

Клиентская часть (Frontend):
- Структура: HTML5
- Стилизация: CSS3 (современная темная тема, использование переменных и Flexbox/Grid)
- Логика: Vanilla JavaScript (чистый JS без фреймворков)
- Архитектура: SPA (Single Page Application) с динамической подгрузкой контента

ОПИСАНИЕ ФУНКЦИОНАЛЬНОЙ АРХИТЕКТУРЫ

Система построена по классической клиент-серверной архитектуре с полным разделением ответственности:

1. Backend (API Layer):
Обеспечивает RESTful интерфейс для всех операций. Реализует бизнес-логику, проверку прав доступа и взаимодействие с базой данных. Включает в себя механизмы миграции и инициализации схемы данных.

2. Frontend (UI Layer):
Представляет собой легковесное веб-приложение. Вся логика отображения, навигации между разделами и взаимодействия с API сосредоточена в JavaScript-модулях (app.js, api.js). Обмен данными происходит в формате JSON.

3. Уровень данных:
Используется реляционная модель данных, реализованная в SQLite. SQLAlchemy обеспечивает абстракцию над SQL-запросами и автоматизирует работу с таблицами.

ФУНКЦИОНАЛЬНЫЕ МОДУЛИ

1. Модуль авторизации (Auth):
Управление учетными записями пользователей. Обработка регистрации, входа в систему и поддержка защищенных сессий через JWT-токены.

2. Дашборд и Аналитика (Reports):
Центральный пульт управления, отображающий ключевые метрики бизнеса: общая выручка, количество заказов, задолженность, количество клиентов и товаров. Сводные таблицы лидеров продаж.

3. Управление клиентами (Clients):
CRM-функционал для ведения базы контрагентов. Хранение контактных данных, информации о компаниях, телефонах и городах. Поддержка создания, редактирования и удаления записей.

4. Управление товарами (Products):
Каталог продукции компании. Учет номенклатуры, артикулов (SKU), цен и актуальных остатков на складе.

5. Управление заказами (Orders):
Центральный модуль системы, связывающий клиентов и товары. Отслеживание статусов заказов (Новый, В работе, Отгружен и т.д.), расчет общей стоимости и контроль дебиторской задолженности.

6. Учет платежей (Payments):
Финансовый контур системы. Регистрация оплат, предоплат и возвратов. Мониторинг статусов транзакций и их связь с конкретными заказами.
